📚 LLM Gateway V10 · Help

Dashboard Help

LLM Gateway V10 routes chat, vision, and embedding calls across eight worker providers, with automatic failover, a separate router pool for auto_route tier classification, rate-limit tracking, per-agent cost ledgers, and persistent logging. Default port: 8110.

What's new in V10

  • Vision — POST /v1/vision and multimodal /v1/chat (image blocks auto-route to vision-capable models)
  • Embeddings — POST /v1/embed with fixed 768-dim vectors (Ollama → OpenAI/Gemini failover)
  • Agent tagging — pass agent and session on chat calls; roll up cost via /v1/cost/by_agent
  • Batch — POST /v1/chat/batch for bounded-concurrency multi-call dispatch
  • Router pool — auto_route=perception|memory|decision picks TINY / LARGE / HUGE tier before worker failover

Using the gateway

Python (built-in client)

from client import LLM, ask

# one-shot
print(ask("Explain transformers in 3 lines"))

# explicit provider (shortcut keys: oai, g, n, o, gr, c, or, gh)
print(ask("hi", provider="gr"))   # groq
print(ask("hi", provider="g"))    # gemini

# full client with agent/session tagging (V10)
llm = LLM()  # defaults to http://localhost:8110
r = llm.chat("hi", provider="gemini", agent="planner", session="my-run",
              max_tokens=128, temperature=0.5)
print(r["text"], r["provider"], r["latency_ms"])

# auto-routed call
r = llm.chat("Summarize this passage…", auto_route="memory")

# embedding (768-dim)
vec = llm.embed("hello world", task_type="retrieval_document")

# streaming
for chunk in llm.stream("count to 5"):
    print(chunk, end="", flush=True)

curl http://localhost:8110/v1/chat \

Provider overview

1. OpenAI

  • Default: gpt-4o-mini ★ — tools, structured output, vision
  • Also used as embedding fallback: text-embedding-3-small (768-dim when configured)
  • Paid list rates; set OPENAI_API_KEY in .env

2. Gemini (Google AI Studio)

  • Best models: gemini-3.1-pro-preview, gemini-3.1-flash-lite ★ default, gemini-2.5-flash
  • Context: 1M tokens in / 65K out · vision-capable
  • Free tier (Flash-Lite): ~15 RPM · 250K TPM · 1,000 RPD

Get a key: aistudio.google.com/app/apikey → Create API key (no card required for free tier).

3. NVIDIA NIM (build.nvidia.com)

  • Default: deepseek-ai/deepseek-v3.2 ★
  • Free tier: 40 RPM, 1,000 inference credits on signup

Get a key: build.nvidia.com → any model → Get API Key.

4. Groq fastest cloud

  • Worker default: openai/gpt-oss-120b ★ (reasoning knob)
  • Router model: llama-3.3-70b-versatile
  • Free tier: 30 RPM · 6,000 TPM · 1,000 RPD

Get a key: console.groq.com → API Keys.

5. Cerebras 2000+ tok/s

  • Worker default: zai-glm-4.7 ★
  • Router default: llama3.1-8b (deprecates May 2026 — override via ROUTER_CEREBRAS_MODEL)
  • Free tier: 30 RPM · 1M tokens/day · 8K ctx cap

Get a key: cloud.cerebras.ai → API Keys.

6. OpenRouter one key, many models

  • Default: nvidia/nemotron-3-super-120b-a12b:free ★
  • Free tier: 20 RPM · 50 RPD per free model

Get a key: openrouter.ai → Keys.

7. GitHub Models

  • Default: openai/gpt-4.1-mini ★
  • Free tier per-request cap: 8K input / 4K output tokens
  • Low-tier: 15 RPM · 150 RPD · High-tier: 10 RPM · 50 RPD

Get a key: github.com/marketplace/models → Use this model → Get API key.

8. Ollama (local)

  • Runs on your machine — zero rate limits, zero cost
  • Chat model via OLLAMA_MODEL; embeddings via nomic-embed-text (768-dim)

Set up: ollama.com → ollama pull nomic-embed-text for embeddings.

Configuration (.env)

The gateway reads .env beside the gateway directory or one level up. All keys are optional — unconfigured providers are skipped.

OPENAI_MODEL=gpt-4o-mini

GEMINI_MODEL=gemini-3.1-flash-lite

How failover works

  1. Caller sends a request (optional provider, auto_route, agent).
  2. If auto_route is set and no explicit provider: router pool classifies TINY / LARGE / HUGE, then worker order follows the tier table.
  3. Router estimates token usage; skips providers in cooldown or over RPM/RPD/TPM caps.
  4. First eligible worker gets the call. On 5xx / timeout / 429 → next provider (unless provider was pinned).
  5. Every call is logged to SQLite with call_role (worker, router_*, embed) and optional agent / session tags.
